UEMySQL 插件使用文档

适用引擎: Unreal Engine 4.27  |  平台: Windows 64 位  |  依赖库: MySQL Connector/C 6.1（libmysql） |  接口: 纯 C++

UEMySQL 是一个用于在 UE4.27 中连接 MySQL 数据库的运行时插件。它封装了 MySQL 官方 C 客户端库 libmysql，提供异步执行、连接池、预处理语句防注入三大能力，并在连接的每个环节统一使用 UTF-8（utf8mb4）编码，彻底解决中文读写乱码问题。

一、插件简介

插件模块名为 UEMySQL，类型为 Runtime，仅在 Win64 平台加载。所有数据库操作都在后台线程池中执行，结果回调会自动切回游戏主线程，避免阻塞渲染与逻辑帧。

核心特性

特性说明对应类
防乱码 UTF-8 全链路连接前后双重设置 utf8mb4，读写自动 UTF-8 ↔ TCHAR 转换FMySQLConnection
防注入 预处理语句? 占位符 + 参数绑定，杜绝 SQL 注入FMySQLConnection::ExecutePrepared
连接池 线程安全复用启动即建立 N 个连接，RAII 借还，失效自动重建FMySQLConnectionPool
异步 不卡主线程后台线程执行，回调切回游戏线程；支持 TFutureFMySQLAsyncExecutor

二、环境准备

插件本身不包含 MySQL 客户端库（体积与授权原因），你需要下载 MySQL Connector/C 6.1 并把三类文件放到 ThirdParty/MySQL/。

下载 MySQL Connector/C 6.1

1 获取连接库

  1. 访问 MySQL 官方下载页：https://downloads.mysql.com/archives/c-c/
  2. Product Version 选择 6.1.11，Operating System 选择 Microsoft Windows，架构选择 64-bit。
  3. 下载 mysql-connector-c-6.1.11-winx64.zip（免安装 zip 版即可）。
  4. 解压后可看到 include/、lib/、bin/ 三个目录。
为什么用 6.1？ 6.1 版是最经典、UE4 教程最多、编译兼容性最好的纯 C 客户端库（编译期无 C++ ABI / OpenSSL 版本冲突），适合 UE4.27（VS2019）。若你使用 MySQL 8.0 自带的 libmysql 也可，但要确保 VC 运行库版本匹配。

放置到 ThirdParty 目录

按下表把解压出来的文件复制到插件的 ThirdParty/MySQL/ 下（目录已在 Build.cs 中约定好）：

来源（Connector 解压后）目标路径说明
include/*ThirdParty/MySQL/include/头文件，含 mysql.h
lib/libmysql.libThirdParty/MySQL/lib/libmysql.lib导入库（链接期）
lib/libmysql.dll 或 bin/libmysql.dllThirdParty/MySQL/bin/libmysql.dll运行时 DLL

放置完成后，目录应如下所示：

ThirdParty/MySQL/
├── include/
│   ├── mysql.h
│   ├── mysql_com.h
│   └── ... # 其余头文件
├── lib/
│   └── libmysql.lib
└── bin/
    └── libmysql.dll
校验位数：务必使用 64 位的 libmysql.lib / libmysql.dll，与 UE4 Win64 一致；混用 32 位会在链接时报 module machine type conflicts。

三、编译与启用

1 放入插件

  1. 确认整个 UEMySQL 文件夹位于工程的 Plugins/ 目录下。
  2. 确认第二章的 ThirdParty/MySQL/ 文件已就位。

2 重新生成工程文件

  1. 右键 .uproject → Generate Visual Studio project files。
  2. 用 Visual Studio 打开 .sln，选择 Development Editor / Win64。

3 编译并确认加载

  1. 编译工程（或直接启动编辑器时自动编译）。
  2. 在 输出日志中过滤 LogUEMySQL，看到 libmysql.dll 加载成功 即表示插件就绪。
  3. 依赖模块只需在你自己模块的 Build.cs 中加入 "UEMySQL"：
csPublicDependencyModuleNames.AddRange(new string[] {
    "Core", "CoreUObject", "Engine",
    "UEMySQL" // 添加本插件模块
});

字段类型默认值说明
HostFString127.0.0.1数据库地址
Portint323306端口（与面板"启动端口"一致）
UserFStringroot用户名
PasswordFString""密码
DatabaseFString""默认库名
CharsetNameFStringutf8mb4防乱码 连接字符集
ConnectTimeoutSecondsint325连接超时（秒）
bAutoReconnectbooltrue断线自动重连

你数据库面板里字符集若为 utf8，插件的 utf8mb4 完全兼容读写；utf8mb4 还额外支持 emoji。若表本身是 utf8，无需改动。

结果结构体 FMySQLResult

字段类型说明
bSuccessbool是否成功
ErrorCode / ErrorMessageint32 / FString失败时的错误码与信息
ColumnNamesTArray<FString>列名（SELECT）
RowsTArray<FMySQLRow>结果行，Row.Get("列名") 取值
AffectedRowsint64受影响行数（增删改）
LastInsertIdint64自增 ID（INSERT）

FMySQLRow 提供 Get(Column) 取字符串值、IsNull(Column) 判断是否为 NULL。

异步执行器 FMySQLAsyncExecutor

方法说明
Start(Config, OutError)创建并初始化连接池
Stop()关闭连接池
ExecuteAsync(SQL, OnComplete)异步执行原始 SQL，回调在游戏线程
ExecutePreparedAsync(SQL, Params, OnComplete)推荐 异步预处理，防注入
ExecutePreparedFuture(SQL, Params)返回 TFuture<FMySQLResult>
ExecutePreparedSync(SQL, Params)同步执行（阻塞当前线程）

参数用 FMySQLParam 工厂函数构造：MakeInt / MakeDouble / MakeString / MakeBlob / MakeNull。

五、使用示例

下面示例通常放在 GameInstance、GameMode 或某个管理器 Actor 中。执行器建议作为长生命周期成员持有。

初始化连接

cpp#include "UEMySQLAsyncExecutor.h"

// 作为成员变量持有（.h）
FMySQLAsyncExecutorPtr DBExecutor;

// 初始化（.cpp，例如 UGameInstance::Init）
void InitDatabase()
{
    FMySQLConnectionConfig Cfg;
    Cfg.Host     = TEXT("127.0.0.1");
    Cfg.Port     = 3306;                // 与面板"启动端口"一致
    Cfg.User     = TEXT("root");
    Cfg.Password = TEXT("123456");
    Cfg.Database = TEXT("game_db");
    Cfg.CharsetName = TEXT("utf8mb4"); // 防乱码，默认即此值
    Cfg.PoolSize    = 4;

    DBExecutor = MakeShared<FMySQLAsyncExecutor, ESPMode::ThreadSafe>();
    FString Err;
    if (!DBExecutor->Start(Cfg, Err))
    {
        UE_LOG(LogTemp, Error, TEXT("MySQL 连接失败: %s"), *Err);
    }
}

查询与增删改（预处理防注入）

查询（SELECT）——用户名含中文也不会乱码：

cppTArray<FMySQLParam> Params;
Params.Add(FMySQLParam::MakeString(TEXT("张三")));   // 中文参数
Params.Add(FMySQLParam::MakeInt(18));

DBExecutor->ExecutePreparedAsync(
    TEXT("SELECT id, name, city FROM users WHERE name=? AND age>?"),
    Params,
    FMySQLResultDelegate::CreateLambda([](const FMySQLResult& R)
    {
        // 回调已在游戏线程，可安全操作 UObject / UI
        if (!R.bSuccess)
        {
            UE_LOG(LogTemp, Error, TEXT("查询失败 [%d]: %s"), R.ErrorCode, *R.ErrorMessage);
            return;
        }
        for (const FMySQLRow& Row : R.Rows)
        {
            FString Id   = Row.Get(TEXT("id"));
            FString Name = Row.Get(TEXT("name"));  // 中文正确显示
            FString City = Row.Get(TEXT("city"));
            UE_LOG(LogTemp, Log, TEXT("用户 %s = %s (%s)"), *Id, *Name, *City);
        }
    }));

插入（INSERT）——获取自增 ID：

cppTArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeString(TEXT("李四")));
P.Add(FMySQLParam::MakeString(TEXT("北京")));
P.Add(FMySQLParam::MakeInt(20));

DBExecutor->ExecutePreparedAsync(
    TEXT("INSERT INTO users(name, city, age) VALUES(?,?,?)"), P,
    FMySQLResultDelegate::CreateLambda([](const FMySQLResult& R)
    {
        if (R.bSuccess)
            UE_LOG(LogTemp, Log, TEXT("插入成功，新 ID=%lld, 影响 %lld 行"),
                R.LastInsertId, R.AffectedRows);
    }));

TFuture 用法（在自己的后台任务中取结果）

cppTArray<FMySQLParam> P{ FMySQLParam::MakeInt(100) };
TFuture<FMySQLResult> Future =
    DBExecutor->ExecutePreparedFuture(TEXT("SELECT * FROM items WHERE id=?"), P);

// 在后台线程可 Get() 阻塞等待；不要在游戏线程直接 Get() 以免卡帧
FMySQLResult R = Future.Get();
DDL / 无参语句：建表等可用 ExecuteAsync(TEXT("CREATE TABLE ..."), Cb)。凡是拼接了用户输入的语句，一律改用 ExecutePreparedAsync 防注入。

六、防乱码原理

中文乱码的根源是编码不一致：数据库存的是 UTF-8 字节，而客户端却按 GBK 或 Latin1 解释。本插件从三个环节强制统一 UTF-8：

1 连接前：协商字符集

在 mysql_real_connect 之前调用 mysql_options(MYSQL_SET_CHARSET_NAME, "utf8mb4")，握手阶段就告诉服务器客户端用 UTF-8。

2 连接后：显式 SET NAMES

连接成功后再调用 mysql_set_character_set(conn, "utf8mb4")（等价于 SET NAMES utf8mb4），双保险统一 client / connection / results 三个字符集。

3 读写：字节级转换

写入时把 FString（UTF-16）经 FTCHARToUTF8 转 UTF-8 字节交给 C API；读取时按 mysql_fetch_lengths 返回的字节长度用 FUTF8ToTCHAR 还原，正确处理中文多字节，绝不截断。

cpp// 连接前
mysql_options(Connection, MYSQL_SET_CHARSET_NAME, "utf8mb4");
// 连接后
mysql_set_character_set(Connection, "utf8mb4");
// 读取：按字节长度转换（关键，避免中文被切半）
FUTF8ToTCHAR Conv(RowData[i], Lengths[i]);
前提：数据库表和字段本身应为 utf8 或 utf8mb4 排序规则。若历史表是 latin1 存了中文，需先用 ALTER TABLE ... CONVERT TO CHARACTER SET utf8mb4 修正，插件无法挽救已损坏的历史数据。

七、常见问题 FAQ

现象原因与解决
启动报"找不到 libmysql.dll"确认 ThirdParty/MySQL/bin/libmysql.dll 存在且为 64 位。打包后 DLL 会自动拷到 Binaries/Win64。
链接错误 machine type conflicts用了 32 位的 lib。必须使用 winx64 版本。
连接报 2003 Can't connect数据库未启动，或端口/防火墙问题。核对面板"启动端口"与 Port。
连接报 1045 Access denied用户名/密码错误，或该用户无远程访问权限。
中文仍乱码确认表字段是 utf8/utf8mb4；老数据是 latin1 存的需先转换字符集。
回调里能操作 UI 吗可以。ExecuteXxxAsync 的回调保证在游戏线程执行。
游戏线程能同步查吗可以用 ExecutePreparedSync，但会阻塞当前帧，仅建议用于加载阶段或后台线程。
小结：拷入 ThirdParty/MySQL → 生成工程 → 编译 → 持有 FMySQLAsyncExecutor 并 Start → 用 ExecutePreparedAsync 读写。全程 utf8mb4，中文无忧，参数绑定防注入，连接池 + 异步不卡主线程。
