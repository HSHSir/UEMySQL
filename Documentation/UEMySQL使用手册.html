UEMySQL 插件使用手册

基于 MySQL Connector/C 6.1（libmysql）的 UE4.27 数据库连接插件 · 异步执行 / 连接池 / 预处理防注入 / 全链路 UTF-8

Database 异步执行 连接池 预处理防注入 UTF-8 仅 Win64 无蓝图节点

1. 概述

UEMySQL 是 NetGame 自研、面向 Unreal Engine 4.27 的 MySQL 数据库访问插件，底层基于 MySQL Connector/C 6.1（libmysql，客户端库版本 6.1.11，对应服务端协议 5.7.16）。它把 C API 封装为面向游戏服务的现代 C++ 接口，提供三大能力支柱：

  1. 异步执行：SQL 在后台线程池执行，结果回调默认切回游戏线程（可直接操作 UObject/UI）。
  2. 线程安全连接池：启动时预建 N 个连接，RAII 借还，失效自动重建。
  3. 预处理语句（Prepated Statement）：? 占位符 + 参数绑定，从根源上防止 SQL 注入。
重要前提：UEMySQL 当前为纯 C++ 插件，所有核心类均为普通 C++ 类（非 UObject），没有任何 BlueprintCallable 蓝图节点。蓝图集成需自行封装一层 UObject 转发（见 §12）。平台仅支持 Win64。

2. 核心特性

特性说明
异步执行 + 回游戏线程后台线程池执行，AsyncTask(ENamedThreads::GameThread) 切回主线程回调
线程安全连接池固定大小池，Acquire 阻塞借出 / Release 归还前存活检测与重建
预处理语句防注入FMySQLParam 工厂函数绑定 Null/Int64/Double/String/Blob
全链路 UTF-8连接前协商 utf8mb4 + 连接后 SET NAMES + 字节级转换，三重防乱码
断线自动重连MYSQL_OPT_RECONNECT，IsAlive() 触发重连
连接超时配置MYSQL_OPT_CONNECT_TIMEOUT
延迟加载 DLLDLL 缺失时程序仍可启动（仅 MySQL 不可用），PublicDelayLoadDLLs
TFuture 支持ExecutePreparedFuture 返回 TFuture<FMySQLResult>

3. 环境准备

3.1 下载 MySQL Connector/C

第三方库因体积与授权未入库，需手动下载 MySQL Connector/C 6.1.11（Microsoft Windows, 64-bit）：

# 官方归档地址
https://downloads.mysql.com/archives/c-c/

# 下载文件
mysql-connector-c-6.1.11-winx64.zip
务必 64 位：若误用 32 位库，链接期将报 module machine type conflicts（x86 与 x64 冲突）。

3.2 放置第三方库

解压后，将头文件与库文件放入插件目录对应位置（路径由 UEMySQLCore.Build.cs 引用）：

Engine/Plugins/HSH/UEMySQL/ThirdParty/MySQL/
├── include/        # 来自 zip 的 include/（含 mysql.h, mysql_version.h 等）
├── lib/
│   └── libmysql.lib # 导入库（链接期）
└── bin/
    └── libmysql.dll # 运行时 DLL（64 位）

3.3 libmysql.dll 部署

DLL 部署由 UEMySQLCore.Build.cs 自动处理：

  • 打包时通过 RuntimeDependencies.Add("$(BinaryOutputDir)/libmysql.dll", Dll) 自动拷贝到输出目录。
  • 模块启动（FUEMySQLModule::StartupModule）还会按以下候选路径显式加载，优先级从高到低：
    1. 插件目录 ThirdParty/MySQL/bin/libmysql.dll
    2. 项目目录/libmysql.dll（打包后 .exe 同目录）
    3. 项目目录/Binaries/Win64/libmysql.dll
    4. 引擎目录/Binaries/Win64/libmysql.dll
  • 延迟加载（PublicDelayLoadDLLs）保证 DLL 缺失时程序仍能启动，仅 MySQL 功能不可用。

7. 主要接口（API）

7.1 FMySQLConnection（单连接 RAII 封装）

非线程安全，由连接池保证独占。类型别名 FMySQLConnectionPtr = TSharedPtr<FMySQLConnection, ESPMode::ThreadSafe>。

方法说明
bool Connect(const FMySQLConnectionConfig&, FString& OutError)建立连接
void Close()关闭连接
bool IsAlive()mysql_ping，自动重连开启时会重连
FMySQLResult Execute(const FString& SQL)直接执行（无参数，谨慎注入）
FMySQLResult ExecutePrepared(const FString& SQL, const TArray<FMySQLParam>& Params)预处理执行（防注入）
FString EscapeString(const FString&) const兜底转义（建议用预处理替代）

7.2 FMySQLConnectionPool（线程安全连接池）

方法说明
bool Initialize(const FMySQLConnectionConfig&, FString& OutError)立即建立 PoolSize 个连接
void Shutdown()释放所有连接
FMySQLConnectionPtr Acquire(int32 TimeoutMs = 5000)阻塞借出；≤0 无限等待；超时返回 nullptr
void Release(FMySQLConnectionPtr)归还（含存活检测/重建）
bool IsInitialized() / const FMySQLConnectionConfig& GetConfig()状态查询

RAII 作用域连接 FMySQLScopedConnection：构造借出、析构自动 Release；提供 IsValid()、operator->()、Get()。

7.3 FMySQLAsyncExecutor（异步执行器，推荐入口）

类型别名 FMySQLAsyncExecutorPtr，继承 TSharedFromThis<..., ThreadSafe>，必须用 ThreadSafe 智能指针创建。回调委托 FMySQLResultDelegate。

方法说明
bool Start(const FMySQLConnectionConfig&, FString& OutError)创建并初始化连接池
void Stop()关闭连接池
bool IsRunning()池有效且已初始化
FMySQLConnectionPoolPtr GetPool()暴露底层池（高级手动借还）
void ExecuteAsync(const FString& SQL, FMySQLResultDelegate OnComplete)异步执行（回调在游戏线程）
void ExecutePreparedAsync(SQL, Params, OnComplete)异步预处理（回调在游戏线程）
TFuture<FMySQLResult> ExecutePreparedFuture(SQL, Params)基于 TFuture（勿在游戏线程阻塞 Get）
FMySQLResult ExecuteSync(const FString& SQL)同步阻塞执行
FMySQLResult ExecutePreparedSync(SQL, Params)同步预处理

8. 使用流程与示例

8.1 初始化与连接池

// 必须用 ThreadSafe 模式智能指针（与 TSharedFromThis 一致）
FMySQLAsyncExecutorPtr DBExecutor = MakeShared<FMySQLAsyncExecutor, ESPMode::ThreadSafe>();

FMySQLConnectionConfig Cfg;
Cfg.Host = TEXT("127.0.0.1"); Cfg.Port = 3306;
Cfg.User = TEXT("root"); Cfg.Password = TEXT("123456");
Cfg.Database = TEXT("game_db"); Cfg.CharsetName = TEXT("utf8mb4");
Cfg.PoolSize = 4;

FString Err;
if (!DBExecutor->Start(Cfg, Err))
{
    UE_LOG(LogTemp, Error, TEXT("MySQL 连接池启动失败: %s"), *Err);
}
// 析构/退出时调用 DBExecutor->Stop(); 并 Reset()
智能指针模式陷阱：因 FMySQLAsyncExecutor 继承 TSharedFromThis<..., ThreadSafe>，必须用 MakeShared<..., ESPMode::ThreadSafe>()。使用默认（NotThreadSafe）模式会触发 static_assert 编译失败。同时应使用 TSharedPtr 而非 TSharedRef（需判空与 Reset()）。

8.2 异步查询（回调回游戏线程）

// SELECT，结果回调已在游戏线程，可安全操作 UObject/UI
TArray<FMySQLParam> Params;
Params.Add(FMySQLParam::MakeString(TEXT("张三")));
Params.Add(FMySQLParam::MakeInt(18));

DBExecutor->ExecutePreparedAsync(
    TEXT("SELECT id,name,city FROM users WHERE name=? AND age>?"),
    Params,
    FMySQLResultDelegate::CreateLambda([](const FMySQLResult& R)
    {
        if (!R.bSuccess)
        {
            UE_LOG(LogTemp, Error, TEXT("查询失败 [%d]: %s"), R.ErrorCode, *R.ErrorMessage);
            return;
        }
        for (const FMySQLRow& Row : R.Rows)
        {
            FString Id   = Row.Get(TEXT("id"));
            FString Name = Row.Get(TEXT("name"));
            UE_LOG(LogTemp, Display, TEXT("id=%s name=%s"), *Id, *Name);
        }
    }));

8.3 预处理参数化（防注入）

// INSERT 并取回自增 ID（全部参数化，杜绝拼接 SQL 注入）
TArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeString(TEXT("李四")));
P.Add(FMySQLParam::MakeString(TEXT("北京")));
P.Add(FMySQLParam::MakeInt(20));

DBExecutor->ExecutePreparedAsync(
    TEXT("INSERT INTO users(name,city,age) VALUES(?,?,?)"),
    P,
    [](const FMySQLResult& R)
    {
        if (R.bSuccess)
            UE_LOG(LogTemp, Display, TEXT("新记录 ID=%lld"), R.LastInsertId);
    });

8.4 TFuture 用法

TFuture<FMySQLResult> Future = DBExecutor->ExecutePreparedFuture(
    TEXT("SELECT COUNT(*) AS cnt FROM users"), {});

// 在工作线程中等待，勿在游戏线程直接 .Get() 阻塞
FMySQLResult R = Future.Get();
不要在游戏线程阻塞：TFuture::Get() 会阻塞当前线程。游戏线程中请使用 ExecutePreparedAsync 回调方式。

8.5 同步用法

适合服务端主/工作线程（如 HSHCustomServer 主循环），不会阻塞游戏线程：

FMySQLResult All = DBExecutor->ExecuteSync(TEXT("SELECT id,name FROM users ORDER BY id"));
if (All.bSuccess)
    for (const FMySQLRow& Row : All.Rows)
        UE_LOG(LogTemp, Display, TEXT("id=%s name=%s"), *Row.Get(TEXT("id")), *Row.Get(TEXT("name")));

8.6 连接池 RAII 直接用法

// 直接使用连接池 + 作用域连接（自动借还）
FMySQLConnectionPoolPtr Pool = DBExecutor->GetPool();
if (Pool && Pool->IsInitialized())
{
    Pool->Initialize(Cfg, Err); // 或用已 Start 的执行器池
    {
        FMySQLScopedConnection Conn(Pool);
        if (Conn.IsValid())
            Conn->ExecutePrepared(TEXT("UPDATE users SET age=? WHERE id=?"),
                                  {FMySQLParam::MakeInt(20), FMySQLParam::MakeInt(1)});
    } // 析构自动归还
}

8.7 事务与存储过程

无事务 API：插件未提供 BeginTransaction/Commit/Rollback 封装。需用原生语句在同一连接连续性上操作：
// 事务（注意：须在“同一连接”上连续执行，建议用同步接口或手动借连接）
DBExecutor->ExecuteSync(TEXT("START TRANSACTION"));
FMySQLResult R1 = DBExecutor->ExecutePreparedSync(TransferSQL, P1);
if (R1.bSuccess)
    DBExecutor->ExecuteSync(TEXT("COMMIT"));
else
    DBExecutor->ExecuteSync(TEXT("ROLLBACK"));

// 存储过程（mysql_real_connect 已带 CLIENT_MULTI_STATEMENTS）
DBExecutor->ExecuteSync(TEXT("CALL my_proc(1, 2)"));
多结果集：当前 ParseStoreResult 仅取第一个结果集；存储过程返回多结果集/OUT 参数需自行扩展。

8.8 在 HSHCustomServer 中集成

服务端控制台程序使用同步接口（主循环中直接执行）。节选自 HSHCustomServer.cpp：

FMySQLConnectionConfig DbConfig;
DbConfig.Host = TEXT("127.0.0.1"); DbConfig.Port = 3306;
DbConfig.User = TEXT("root"); DbConfig.Password = TEXT("root");
DbConfig.Database = TEXT("test"); DbConfig.PoolSize = 4;

FMySQLAsyncExecutorPtr DbExecutor = MakeShared<FMySQLAsyncExecutor, ESPMode::ThreadSafe>();
FString DbError;
if (DbExecutor->Start(DbConfig, DbError))
{
    // ping
    FMySQLResult Ping = DbExecutor->ExecutePreparedSync(
        TEXT("SELECT 1 + ? AS result"), {FMySQLParam::MakeInt(41)});
    if (Ping.bSuccess && Ping.Rows.Num() > 0)
        UE_LOG(LogTemp, Display, TEXT("ping ok, 1+41 = %s"), *Ping.Rows[0].Get(TEXT("result")));

    // 参数化条件查询
    FMySQLResult CityUsers = DbExecutor->ExecutePreparedSync(
        TEXT("SELECT id,name,age FROM uemysql_test_users WHERE city = ? ORDER BY id"),
        {FMySQLParam::MakeString(TEXT("武当山"))});
}
// 退出时：
DbExecutor->Stop();
DbExecutor.Reset();

8.9 完整示例

// Copyright NetGame. All Rights Reserved.
//
// HSHCustomServer —— 基于 UE4 源码的自定义服务器程序。
//
// 本程序以 Engine/Source/Programs/BlankProgram 为骨架，接入两个位于
// Engine/Plugins/HSH 下的自研插件：
//   1) HighCapacityNet —— 高性能并发网络连接（TCP/KCP 服务端，支持 5w+ 连接）
//   2) UEMySQL         —— 基于 MySQL Connector/C 的数据库连接（连接池 / 预处理防注入）
//
// 程序启动后：
//   - 显式加载两个插件模块；
//   - 启动 HighCapacityNet 的 TCP 监听（默认 0.0.0.0:7777）；
//   - 初始化 UEMySQL 连接池，并演示一次同步查询；
//   - 主循环中每帧驱动 NetConnectionManager::Tick，维持连接与收发。
//
// 注意：本程序为无界面（headless）控制台服务，不依赖 World / GameMode，
// 由 main 函数本身充当“GameMode”角色驱动网络层 Tick。

#include "HSHCustomServer.h"

#include "RequiredProgramMainCPPInclude.h"

#include "Modules/ModuleManager.h"

// HighCapacityNet 插件（底层核心模块）
#include "NetConnectionManager.h"     // UNetConnectionManager（网络层主入口 UObject）

// UEMySQL 插件（底层核心模块）
#include "UEMySQLAsyncExecutor.h"     // FMySQLAsyncExecutor（含同步接口）

// 修复 Windows 控制台中文乱码：将控制台代码页切换为 UTF-8（CP=65001）
// UE4 内部字符串为 UTF-16，默认 GBK(CP936) 控制台输出中文会变问号。
#include "Windows/MinWindows.h"

DEFINE_LOG_CATEGORY_STATIC(LogHSHCustomServer, Log, All);

IMPLEMENT_APPLICATION(HSHCustomServer, "HSHCustomServer");

INT32_MAIN_INT32_ARGC_TCHAR_ARGV()
{
	// =========================================================================
	// 【关键】必须在 PreInit() 之前设置控制台 UTF-8！
	//
	// UE4 的 FOutputDeviceStdOutput 构造函数(LaunchEngineLoop.cpp:328)在 PreInit
	// 内部被执行，它会检测 stdout 是否连接到控制台(IsStdoutAttachedToConsole)
	// 来决定日志输出路径：
	//   - bIsConsoleOutput=true  → WriteConsoleW (Unicode/UTF-16 直写，不受代码页影响)
	//   - bIsConsoleOutput=false → printf(TCHAR_TO_UTF8) (受 _setmode + 控制台代码页影响)
	// 无论走哪条路径，控制台代码页都必须在 PreInit 之前设为 UTF-8，
	// 且 SetUTF8Output() 确保 C 运行时 stdout 模式正确。
	// =========================================================================
	SetConsoleOutputCP(65001);   // 控制台输出 → UTF-8 代码页
	SetConsoleCP(65001);         // 控制台输入 → UTF-8 代码页

	// 1) 引擎预初始化：建立 Core / CoreUObject / Engine、模块管理器，
	//    并自动加载 EnabledByDefault 的引擎插件。
	GEngineLoop.PreInit(ArgC, ArgV);

	// PreInit 后确保 C 运行时 stdout 为 UTF-8 文本模式
	// （与 WindowsPlatformMisc::SetUTF8Output 一致：_setmode(stdout, _O_U8TEXT)）
	FPlatformMisc::SetUTF8Output();

	// 2) 显式加载两个 HSH 插件的“底层核心模块”（初始化内存池 / 网络线程 / 监控，
	//    以及加载 libmysql.dll）。即便已随 PreInit 自动加载，此处调用也是幂等的。
	FModuleManager::Get().LoadModuleChecked("HighCapacityNetCore");
	FModuleManager::Get().LoadModuleChecked("UEMySQLCore");

	UE_LOG(LogHSHCustomServer, Display, TEXT("=================================================="));
	UE_LOG(LogHSHCustomServer, Display, TEXT(" HSHCustomServer starting..."));
	UE_LOG(LogHSHCustomServer, Display, TEXT("=================================================="));

	// -------------------------------------------------------------------------
	// 3) 网络层：启动 HighCapacityNet TCP 服务端
	// -------------------------------------------------------------------------
	UNetConnectionManager* NetMgr = UNetConnectionManager::CreateInstance(nullptr);
	NetMgr->Initialize(ENetServerRole::Standalone);

	// 注册一个示例消息处理器（消息类型 >= 1000 为用户消息）。
	NetMgr->RegisterMessageHandler(1001, FNetMessageHandler::CreateLambda(
		[](int32 ConnId, int32 MsgType, TSharedPtr Data)
		{
			UE_LOG(LogHSHCustomServer, Display,
				TEXT("[Net] recv msgType=%d from conn=%d (%s)"),
				MsgType, ConnId, Data.IsValid() ? TEXT("has payload") : TEXT("empty"));
			// 业务示例：在此解析 Data 中的协议，并调用下方 MySQL 接口读写数据库。
		}));

	const int32 ListenPort = 7777;
	if (NetMgr->StartTcpServer(TEXT("0.0.0.0"), ListenPort, 10000))
	{
		UE_LOG(LogHSHCustomServer, Display, TEXT("[Net] TCP server listening on 0.0.0.0:%d"), ListenPort);
	}
	else
	{
		UE_LOG(LogHSHCustomServer, Warning, TEXT("[Net] failed to start TCP server on port %d"), ListenPort);
	}

	// -------------------------------------------------------------------------
	// 4) 数据库层：初始化 UEMySQL 连接池（使用同步接口，适合服务器主/工作线程）
	// -------------------------------------------------------------------------
	FMySQLConnectionConfig DbConfig;
	DbConfig.Host = TEXT("127.0.0.1");
	DbConfig.Port = 3306;
	DbConfig.User = TEXT("root");
	DbConfig.Password = TEXT("root");
	DbConfig.Database = TEXT("test");
	DbConfig.PoolSize = 4;

	// FMySQLAsyncExecutor 继承自 TSharedFromThis<..., ESPMode::ThreadSafe>，
	// 必须用 ThreadSafe 模式的智能指针（官方别名 FMySQLAsyncExecutorPtr），
	// 否则 MakeShared 默认 NotThreadSafe 模式会与 TSharedFromThis 模式冲突 → static_assert。
	// 用 TSharedPtr 而非 TSharedRef：下方退出逻辑需要 if(DbExecutor) 判空与 Reset()，
	// 二者均为 TSharedPtr 的接口；TSharedRef 永不为空且无 Reset()，会导致编译错误。
	FMySQLAsyncExecutorPtr DbExecutor = MakeShared();
	FString DbError;
	if (DbExecutor->Start(DbConfig, DbError))
	{
		UE_LOG(LogHSHCustomServer, Display, TEXT("[DB] MySQL connection pool started (pool size=%d)"), DbConfig.PoolSize);

		// ---- 示例 A：ping 验证连通性 ----
		TArray PingParams;
		PingParams.Add(FMySQLParam::MakeInt(41));
		FMySQLResult Ping = DbExecutor->ExecutePreparedSync(TEXT("SELECT 1 + ? AS result"), PingParams);

		if (Ping.bSuccess && Ping.Rows.Num() > 0)
		{
			UE_LOG(LogHSHCustomServer, Display, TEXT("[DB] ping ok, 1+41 = %s"), *Ping.Rows[0].Get(TEXT("result")));
		}
		else
		{
			UE_LOG(LogHSHCustomServer, Warning, TEXT("[DB] ping failed: %s"), *Ping.ErrorMessage);
		}


		// B1) 全量查询：SELECT * FROM users
		{
			FMySQLResult AllUsers = DbExecutor->ExecuteSync(TEXT("SELECT id, name, city, age FROM uemysql_test_users ORDER BY id"));
			if (AllUsers.bSuccess)
			{
				UE_LOG(LogHSHCustomServer, Display, TEXT("[DB] === users 全量查询 (%d 条) ==="), AllUsers.Rows.Num());
				for (const FMySQLRow& Row : AllUsers.Rows)
				{
					UE_LOG(LogHSHCustomServer, Display,
						TEXT("[DB]   id=%s  name=%s  city=%s  age=%s"),
						*Row.Get(TEXT("id")), *Row.Get(TEXT("name")),
						*Row.Get(TEXT("city")), *Row.Get(TEXT("age")));
				}
			}
			else
			{
				UE_LOG(LogHSHCustomServer, Warning, TEXT("[DB] 全量查询失败: %s"), *AllUsers.ErrorMessage);
			}
		}

		// B2) 条件查询（参数化，防 SQL 注入）：按城市查用户
		{
			TArray CityParams;
			CityParams.Add(FMySQLParam::MakeString(TEXT("武当山")));   // 参数化绑定，防注入
			FMySQLResult CityUsers = DbExecutor->ExecutePreparedSync(
				TEXT("SELECT id, name, age FROM uemysql_test_users WHERE city = ? ORDER BY id"),
				CityParams);

			if (CityUsers.bSuccess)
			{
				UE_LOG(LogHSHCustomServer, Display,
					TEXT("[DB] === 城市='武当山' 的用户 (%d 条) ==="), CityUsers.Rows.Num());
				for (const FMySQLRow& Row : CityUsers.Rows)
				{
					UE_LOG(LogHSHCustomServer, Display,
						TEXT("[DB]   id=%s  name=%s  age=%s"),
						*Row.Get(TEXT("id")), *Row.Get(TEXT("name")), *Row.Get(TEXT("age")));
				}
			}
			else
			{
				UE_LOG(LogHSHCustomServer, Warning, TEXT("[DB] 城市条件查询失败: %s"), *CityUsers.ErrorMessage);
			}
		}

		// B3) 聚合统计：各城市的平均年龄
		{
			FMySQLResult Stats = DbExecutor->ExecuteSync(
				TEXT("SELECT city, COUNT(*) AS cnt, AVG(age) AS avg_age FROM uemysql_test_users GROUP BY city ORDER BY cnt DESC"));
			if (Stats.bSuccess)
			{
				UE_LOG(LogHSHCustomServer, Display, TEXT("[DB] === 各城市统计 ==="));
				for (const FMySQLRow& Row : Stats.Rows)
				{
					UE_LOG(LogHSHCustomServer, Display,
						TEXT("[DB]   city=%s  人数=%s  平均年龄=%s"),
						*Row.Get(TEXT("city")), *Row.Get(TEXT("cnt")), *Row.Get(TEXT("avg_age")));
				}
			}
		}

		// B4) 更新示例（参数化）：修改已有用户记录的年龄（幂等，可重复运行）
		{
			TArray UpdateParams;
			UpdateParams.Add(FMySQLParam::MakeInt(100));                    // 新年龄
			UpdateParams.Add(FMySQLParam::MakeString(TEXT("张三")));        // 按姓名定位

			FMySQLResult UpdateRes = DbExecutor->ExecutePreparedSync(
				TEXT("UPDATE uemysql_test_users SET age = ? WHERE name = ?"),
				UpdateParams);

			if (UpdateRes.bSuccess)
			{
				UE_LOG(LogHSHCustomServer, Display,
					TEXT("[DB] 更新成功: 张三年龄改为 100, 影响行数=%d"), UpdateRes.AffectedRows);

				// 回查验证
				FMySQLResult Verify = DbExecutor->ExecutePreparedSync(
					TEXT("SELECT id, name, city, age FROM uemysql_test_users WHERE name = ?"),
					{ FMySQLParam::MakeString(TEXT("张三")) });
				if (Verify.bSuccess && Verify.Rows.Num() > 0)
				{
					const FMySQLRow& R = Verify.Rows[0];
					UE_LOG(LogHSHCustomServer, Display,
						TEXT("[DB]   回查确认: id=%s name=%s city=%s age=%s"),
						*R.Get(TEXT("id")), *R.Get(TEXT("name")), *R.Get(TEXT("city")), *R.Get(TEXT("age")));
				}
			}
			else
			{
				UE_LOG(LogHSHCustomServer, Warning, TEXT("[DB] 更新失败: %s"), *UpdateRes.ErrorMessage);
			}
		}
	}
	else
	{
		UE_LOG(LogHSHCustomServer, Warning,
			TEXT("[DB] MySQL pool start failed: %s (确认 MySQL 服务已启动且 libmysql.dll 可用)"), *DbError);
	}

	UE_LOG(LogHSHCustomServer, Display, TEXT("HSHCustomServer is running. Press Ctrl+C to stop."));

	// -------------------------------------------------------------------------
	// 5) 主循环：每帧驱动网络层 Tick，并周期性让出 CPU。
	//    HighCapacityNet 的网络 I/O 与连接管理由自身线程 + 本 Tick 共同完成；
	//    UEMySQL 同步接口在业务调用处（如消息处理器内）直接执行。
	// -------------------------------------------------------------------------
	double LastTime = FPlatformTime::Seconds();
	while (!IsEngineExitRequested())
	{
		const double Now = FPlatformTime::Seconds();
		const float DeltaTime = (float)(Now - LastTime);
		LastTime = Now;

		if (NetMgr)
		{
			NetMgr->Tick(DeltaTime);
		}

		FPlatformProcess::Sleep(0.01f); // 约 100Hz 驱动
	}

	// -------------------------------------------------------------------------
	// 6) 优雅退出：先停网络监听与连接，再关闭数据库池，最后走引擎退出流程。
	// -------------------------------------------------------------------------
	UE_LOG(LogHSHCustomServer, Display, TEXT("HSHCustomServer shutting down..."));

	if (NetMgr)
	{
		NetMgr->Shutdown();
		NetMgr = nullptr;
	}

	if (DbExecutor)
	{
		DbExecutor->Stop();
		DbExecutor.Reset();
	}

	FEngineLoop::AppExit();
	return 0;
}

9. 防中文乱码原理

UEMySQL 采用三重防护确保中文与 emoji 不乱码：

阶段措施代码位置
① 连接前协商mysql_options(..., MYSQL_SET_CHARSET_NAME, "utf8mb4")FMySQLConnection::Connect
② 连接后 SET NAMESmysql_set_character_set(conn, "utf8mb4")（等价 SET NAMES）FMySQLConnection::ApplyCharset
③ 字节级转换FString（UTF-16）↔ UTF-8 字节，绑定参数时转 UTF-8，取回时 UTF-8→TCHAR参数绑定 / 结果解析

务必保持数据库/表/列的字符集为 utf8mb4，且 FMySQLConnectionConfig::CharsetName 默认即为 utf8mb4。

11. 注意事项与已知限制

  • 无蓝图节点：当前无任何 BlueprintCallable；所有类为普通 C++ 类。UEMySQL 模块为占位，蓝图暴露为 TODO（见 §12）。
  • 无 ini 配置：配置只能硬编码在 FMySQLConnectionConfig，无可热更的 ini。
  • 无事务 API：无 Commit/Rollback 封装，需用原生 START TRANSACTION/COMMIT/ROLLBACK，并注意同一连接连续性。
  • 无专门存储过程/多结果集 API：ParseStoreResult 只取首个结果集。
  • 列值统一为 FString：数字/日期/小数需调用方自行解析。
  • 参数类型有限：仅 Int64/Double/String/Blob（float 用 Double，二进制用 Blob）。
  • 池不动态扩容：固定 PoolSize，高并发下 Acquire 阻塞至超时（默认 5000ms）。
  • 无每查询超时：仅有连接获取超时（Acquire），无语句级超时。
  • 仅 Win64：非 Win64 平台 WITH_UEMYSQL=0，方法返回错误桩。
  • 单连接非线程安全：不可跨线程共享同一 FMySQLConnection 实例。
  • 智能指针模式：执行器必须用 ESPMode::ThreadSafe 创建（见 §8.1）。

12. 蓝图封装（TODO 说明）

要在蓝图中使用 UEMySQL，需新建一个 UObject 包装类，为关键方法加上 UFUNCTION(BlueprintCallable) 并依赖 Engine 模块。示意骨架：

// 在 UEMySQL 模块（或新模块）中，Build.cs 增加 "Engine" 依赖

public:

13. 常用 SQL 语句用法

本章节汇总在 UEMySQL 下最常用的 SQL 操作范例，均基于前文演示表 uemysql_test_users（字段 id / name / city / age / created_at）。每个示例给出可直接执行的 SQL 与对应的 UEMySQL 调用写法。

参数化优先原则：凡涉及外部输入或变量值，一律用 ExecutePrepared* + ? 占位符（见 §8.3），从根源防止 SQL 注入。表名、列名属于标识符，不能参数化，只能写死或走白名单拼接。DDL（建表/改表）与无变量的语句用 ExecuteSync 直接执行。

13.1 建表（CREATE TABLE）

DDL 不含可变参数，使用 ExecuteSync 直接执行；务必显式指定 DEFAULT CHARSET=utf8mb4 与 InnoDB 引擎，配合 §9 的三重防乱码机制。

// 建表示例（DDL，无需参数化）
const FString CreateSQL = TEXT(R"(
CREATE TABLE IF NOT EXISTS uemysql_test_users (
    id         INT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY,
    name       VARCHAR(64)  NOT NULL DEFAULT '',
    city       VARCHAR(128) NOT NULL DEFAULT '',
    age        INT          NOT NULL DEFAULT 0,
    created_at TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
)");
FMySQLResult R = DBExecutor->ExecuteSync(CreateSQL);
if (R.bSuccess)
    UE_LOG(LogTemp, Display, TEXT("建表成功"));
else
    UE_LOG(LogTemp, Error, TEXT("建表失败 [%d]: %s"), R.ErrorCode, *R.ErrorMessage);
表已存在：生产环境建议用 CREATE TABLE IF NOT EXISTS，避免重复执行报错；或把建表脚本放在部署阶段单独执行，而非每次启动。

13.2 增加数据（INSERT）

单行插入用参数化占位符，并通过 FMySQLResult::LastInsertId 取回自增主键。

// 单行插入（参数化，取回自增 ID）
TArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeString(TEXT("王五")));
P.Add(FMySQLParam::MakeString(TEXT("上海")));
P.Add(FMySQLParam::MakeInt(25));

DBExecutor->ExecutePreparedAsync(
    TEXT("INSERT INTO uemysql_test_users(name, city, age) VALUES(?, ?, ?)"),
    P,
    [](const FMySQLResult& R)
    {
        if (R.bSuccess)
            UE_LOG(LogTemp, Display, TEXT("插入成功，自增 ID=%lld"), R.LastInsertId);
        else
            UE_LOG(LogTemp, Error, TEXT("插入失败: %s"), *R.ErrorMessage);
    });

13.3 查询数据（SELECT）

查询条件一律参数化；结果经回调回游戏线程，按列名逐行取值。数字 / 日期等字段统一以 FString 返回，需调用方自行解析。

// 条件查询（参数化）；回调在游戏线程
TArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeString(TEXT("北京")));
P.Add(FMySQLParam::MakeInt(18));

DBExecutor->ExecutePreparedAsync(
    TEXT("SELECT id, name, city, age FROM uemysql_test_users WHERE city=? AND age > ? ORDER BY id"),
    P,
    [](const FMySQLResult& R)
    {
        if (!R.bSuccess) { UE_LOG(LogTemp, Error, TEXT("查询失败: %s"), *R.ErrorMessage); return; }
        for (const FMySQLRow& Row : R.Rows)
        {
            int32 Id  = FCString::Atoi(*Row.Get(TEXT("id")));
            int32 Age = FCString::Atoi(*Row.Get(TEXT("age")));
            UE_LOG(LogTemp, Display, TEXT("id=%d name=%s city=%s age=%d"),
                   Id, *Row.Get(TEXT("name")), *Row.Get(TEXT("city")), Age);
        }
    });
聚合 / 分页 / 批量读取：COUNT / SUM / AVG / GROUP BY 写法与普通查询一致（参考 §8.9 的 B3 统计示例）。分页用 ... LIMIT ? OFFSET ? 参数化避免拼接页码；按主键集合批量读取用 WHERE id IN (?,?,?)（占位符数量等于 id 个数）。

13.4 修改数据（UPDATE）

// 按主键更新（参数化）；AffectedRows 为受影响行数
TArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeInt(26));                 // 新 age
P.Add(FMySQLParam::MakeString(TEXT("广州")));    // 新 city
P.Add(FMySQLParam::MakeInt(1));                  // WHERE id

FMySQLResult R = DBExecutor->ExecutePreparedSync(
    TEXT("UPDATE uemysql_test_users SET age=?, city=? WHERE id=?"), P);
if (R.bSuccess)
    UE_LOG(LogTemp, Display, TEXT("更新 %d 行"), R.AffectedRows);
漏写 WHERE：UPDATE 若不带 WHERE 会更新整表，UEMySQL 不会拦截。务必用主键或明确条件定位。

13.5 删除数据（DELETE）

// 删除单条（参数化）
FMySQLResult R = DBExecutor->ExecutePreparedSync(
    TEXT("DELETE FROM uemysql_test_users WHERE id=?"),
    { FMySQLParam::MakeInt(1) });
if (R.bSuccess)
    UE_LOG(LogTemp, Display, TEXT("删除 %d 行"), R.AffectedRows);
误删风险：DELETE 务必带 WHERE 条件，缺条件将清空整表。生产环境建议加软删除列 deleted_at，用 UPDATE ... SET deleted_at=NOW() 替代物理删除，便于恢复与审计。

13.6 批量增删改查

UEMySQL 没有原生「批量」API，但可用下列四种模式高效批量操作，按推荐度排序：

（1）多值 INSERT（推荐：单语句、单结果集、网络往返最少）

// 一条预处理语句插入 N 行，参数为「每行字段」平铺
TArray<FMySQLParam> P;
P.Add(FMySQLParam::MakeString(TEXT("a"))); P.Add(FMySQLParam::MakeInt(20));
P.Add(FMySQLParam::MakeString(TEXT("b"))); P.Add(FMySQLParam::MakeInt(21));
P.Add(FMySQLParam::MakeString(TEXT("c"))); P.Add(FMySQLParam::MakeInt(22));

// ? 占位符总数必须等于 行数×列数 = 3 行 × 2 列 = 6 个
DBExecutor->ExecutePreparedSync(
    TEXT("INSERT INTO uemysql_test_users(name, age) VALUES (?,?),(?,?),(?,?)"), P);

（2）IN 子句批量更新 / 删除 / 读取

// 按主键集合批量删除（IN 占位符数量 = id 个数）
TArray<FMySQLParam> Ids;
Ids.Add(FMySQLParam::MakeInt(10));
Ids.Add(FMySQLParam::MakeInt(11));
Ids.Add(FMySQLParam::MakeInt(12));
DBExecutor->ExecutePreparedSync(
    TEXT("DELETE FROM uemysql_test_users WHERE id IN (?,?,?)"), Ids);

// 按主键集合批量读取
DBExecutor->ExecutePreparedSync(
    TEXT("SELECT id,name,city FROM uemysql_test_users WHERE id IN (?,?,?)"), Ids);

（3）循环逐条调用（最简单，适合小批量）

// 遍历业务缓存逐条写入；大批量请优先用方式(1)/(4)以减少往返与事务开销
TArray<FMyUser> Users = GetPendingUsers();
for (const FMyUser& U : Users)
{
    DBExecutor->ExecutePreparedSync(
        TEXT("INSERT INTO uemysql_test_users(name, city, age) VALUES(?,?,?)"),
        { FMySQLParam::MakeString(U.Name),
          FMySQLParam::MakeString(U.City),
          FMySQLParam::MakeInt(U.Age) });
}

（4）LOAD 大批量（进阶）

超大数据量（万级以上）建议导出为 CSV，在数据库侧用 LOAD DATA LOCAL INFILE，比逐行 INSERT 快一个数量级（需服务端开启 local_infile）。UEMySQL 可通过 ExecuteSync 执行该语句。

13.7 事务批量写入（保证一致性）

一批写操作需要「全成功或全回滚」时，用事务包裹。两种写法：

// 方式 A：单条多语句脚本（无外部变量，用 ExecuteSync）
// 本插件已启用 CLIENT_MULTI_STATEMENTS，多个结果集会被自动排空（见修订说明 2026-07-12）
DBExecutor->ExecuteSync(TEXT(R"(
START TRANSACTION;
INSERT INTO uemysql_test_users(name, city, age) VALUES('批量1','北京',30);
INSERT INTO uemysql_test_users(name, city, age) VALUES('批量2','北京',31);
COMMIT;
)"));

// 方式 B：参数化写入 + 手动事务（推荐，同一连接连续性）
DBExecutor->ExecuteSync(TEXT("START TRANSACTION"));
bool bOk = true;
for (const FMyUser& U : GetPendingUsers())
{
    FMySQLResult R = DBExecutor->ExecutePreparedSync(
        TEXT("INSERT INTO uemysql_test_users(name, city, age) VALUES(?,?,?)"),
        { FMySQLParam::MakeString(U.Name),
          FMySQLParam::MakeString(U.City),
          FMySQLParam::MakeInt(U.Age) });
    if (!R.bSuccess) { bOk = false; break; }
}
DBExecutor->ExecuteSync(bOk ? TEXT("COMMIT") : TEXT("ROLLBACK"));
同一连接连续性：事务内所有语句必须落在同一个 FMySQLConnection 上。连接池的每次 ExecuteSync / ExecuteAsync 都可能借到不同连接，使事务失效。正确做法是通过 FMySQLScopedConnection（§8.6）锁定单连接，再在其上执行 START TRANSACTION / 写入 / COMMIT|ROLLBACK。
修订说明（源码同步 · 2026-07-12）：本手册核对源码后，对插件源码做了如下修正与优化，文档内容与此保持一致：
  1. 修正头文件示例：UEMySQLAsyncExecutor 头文件示例代码原为 MakeShared<FMySQLAsyncExecutor>()，已修正为 MakeShared<FMySQLAsyncExecutor, ESPMode::ThreadSafe>()。该类继承自 TSharedFromThis<..., ESPMode::ThreadSafe>，漏写 ESPMode::ThreadSafe 会在运行期触发 AsShared() 断言失败（见 8.1 与注意事项）。
  2. 优化多结果集处理：FMySQLConnection::Execute 在连接启用 CLIENT_MULTI_STATEMENTS 时，原先仅消费第一个结果集，未排空后续结果集（存储过程 / 批量语句会产生多个结果集），会导致后续查询报 Commands out of sync 而损坏连接。已新增 DrainRemainingResults() 排空全部结果集（数据丢弃，仅维持协议同步）。
